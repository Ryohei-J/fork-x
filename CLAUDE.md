# プロジェクト概要

Webエンジニアを対象とした「正社員 vs フリーランス」の収入・税金比較シミュレーター。
年収（または月の報酬）を入力すると、正社員だった場合とフリーランスだった場合の手取り額などを概算で比較表示するWebアプリ。

- 対象ユーザー: Webエンジニア（就業形態の変更を検討している人など）
- 計算は法律に厳密に従う必要はなく、あくまで概算・目安でよい
- 精度向上は少しずつ進めており、現状の簡略化ポイントは [TODO.md](docs/TODO.md) で管理している

## 入力項目

- 正社員の年収／月給、フリーランスの報酬（年間／月額）※それぞれ独立した金額・入力単位で指定可能
- 居住地（都道府県）※現在は参考表示のみ、計算には未反映
- 扶養人数
- 年齢（介護保険料の対象判定に使用）
- フリーランスの必要経費率（%、初期値10%）
- フリーランスの確定申告方式（青色申告／白色申告）

## 出力

- 正社員・フリーランスそれぞれの手取り額と内訳（社会保険料・所得税・住民税など）
- 手取りの差額（多い方を自動判定して表示）

## 技術スタック

- フレームワーク: Next.js（App Router）
- 言語: TypeScript
- パッケージマネージャ: pnpm
- スタイリング: Tailwind CSS
- 計算ロジック: クライアントサイドのみで完結（バックエンドAPIは持たない）
- データ永続化: ブラウザの localStorage のみ（DBやユーザーアカウントは持たない）
- デプロイ: Vercel
- UI言語: 日本語のみ（i18n対応はしない）
- テスト: Vitest（`pnpm test` で実行）。計算ロジックが純粋関数のため、`src/lib/calculator/` 配下にユニットテストを配置している

## 設計方針

- サーバーは持たず、静的ホスティングで完結させる
- 税金・社会保険料の計算ロジックは、日本の制度（所得税・住民税・社会保険料など）をベースにした概算ロジックとして実装し、他のロジックから分離した純粋関数として置く（テストしやすく、将来の税制改正にも対応しやすいように）
- 過不足のある機能や将来対応（多言語化、バックエンドAPI化、DB永続化など）は現時点では実装しない

## 計算ロジックの方針

計算ロジックは `src/lib/calculator/` にまとめる。料率・控除額などの定数は `constants.ts` に分離し、2024年分（令和6年度）の制度をベースにした概算値としてコメント付きで管理する。制度改正時はこのファイルのみ更新すればよい構成にする。

**共通ロジック**（正社員・フリーランス両方で使用、`tax.ts`）
- 所得税（累進課税の速算表 + 復興特別所得税2.1%）
- 住民税（所得割10% + 均等割 概算5,000円、全国一律とみなす）。ただし合計所得金額が非課税限度額（全国共通の概算式）以下の場合は均等割・所得割とも非課税にする
- 基礎控除（48万円）・扶養控除（1人あたり38万円で簡略化）
- 介護保険料の対象年齢判定（40〜64歳）

**正社員側**（`employee.ts`）
- 給与所得控除（速算表）
- 社会保険料は健康保険・厚生年金・雇用保険・介護保険（40〜64歳のみ）を計算
- 健康保険・厚生年金は、年収を「月給12ヶ月分+賞与4ヶ月分（年2回）」という一般的な比率で分解したうえで、標準報酬月額・標準賞与額それぞれの上限を適用する（実際の月給・賞与の内訳は入力させない仮定ベース）

**フリーランス側**（`freelance.ts`）
- 必要経費率はユーザー入力（初期値10%）
- 確定申告方式（青色申告=65万円控除／白色申告=控除なし）をユーザーが選択
- 国民健康保険料は全国平均レートによる概算（所得割+均等割+介護分、賦課限度額106万円でキャップ）。自治体ごとの実際の差異は反映しない
- 国民健康保険料の均等割部分は、低所得世帯向けの軽減措置（7割・5割・2割軽減、全国共通の基準額で判定）を反映する
- 国民年金保険料は定額（本人分のみ、低所得者向けの免除制度は未対応）

**MVPで対象外とするもの**（概算の簡略化のため）
- 個人事業税（業種による対象判定が分かれるため）
- 消費税・インボイス制度（免税事業者を前提）
- 居住地（都道府県）は参考表示のみで、計算には反映しない
- 国民健康保険料・住民税の自治体ごとの実際の料率差（全国平均・全国共通基準で近似）
